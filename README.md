# 🎬 抖音创作者数据分析助手 (Douyin Creator Analytics)

## ✨ 功能说明

### 1. 🔍 无感拦截采集（injected.js）
插件在页面加载时向抖音网页注入一段脚本，同时 Hook（钩挂）浏览器原生的 `XMLHttpRequest` 和 `Fetch` 两种网络请求方式。

- **只抓有效接口**：精准过滤，只拦截 `/aweme/v1/web/aweme/post/`（视频列表）和 `/aweme/v1/web/user/profile/other/`（用户资料）这两类接口，其他接口一律忽略，不影响浏览体验和性能。
- **只抓他人数据**：通过 `other` 关键词严格判断，**不会误采集你自己登录账号的信息**，彻底杜绝了把自己账号抓进数据库的问题。
- **防重复触发**：对非视频 `aweme_list`（例如抖音的用户关注列表、收藏等也包含该字段）进行二次校验，确保只有真正的视频数据才会被写入。

---

### 2. 💾 本地数据库存储（background.js + IndexedDB）
所有数据**只存在你本人电脑的浏览器 IndexedDB 中**，开发者无法访问，也不会上传到任何服务器。

- **accounts 账号表**：存储账号 ID（`sec_uid`）、昵称、粉丝数、历史总获赞数、签名、头像、采集时间。
- **videos 视频表**：存储视频 ID（`aweme_id`）、所属账号 ID、标题、发布时间、视频类型、点赞数、评论数、分享数、收藏数、采集时间。
- **智能 Upsert（更新或插入）机制**：写入视频前先查询该视频 ID 是否已存在。
  - 若是**全新视频** → 插入并计入本次新增数量。
  - 若是**已存在的视频** → 更新其最新点赞、评论等数据，但不重复计数。
- **账号字段保护合并**：针对粉丝数、总获赞、视频数等关键字段，若新数据中该字段为 `0` 或缺失，则自动保留数据库中已有的历史值，防止因接口字段缺失导致数据被覆盖清零。

---

### 3. 📊 本次采集实时计数（popup.js）
弹窗顶部显示的数字代表**本次打开弹窗以来新采集入库的视频条数**，而不是数据库总视频数。

- **每次打开弹窗时自动归零**：确保数字从 0 开始，直观反映"本轮操作"的成果。
- **只计真正的新增**：已在数据库中的旧视频被刷新数据时，不计入此数字，杜绝虚涨。
- **跨账号累加**：本次打开弹窗期间，依次采集多个账号，数字会持续累加，直到你关闭弹窗。

---

### 4. 🤖 自动向下滚动采集（content.js）
点击弹窗中的**"自动向下滚动采集"**按钮后，插件会自动操控浏览器页面向下滚动，无需你手动不停地滚鼠标。

- **智能滚动**：每次滚动后等待网络数据加载完毕，再继续下一次滚动，不会因为滚太快而漏数据。
- **自动停止**：当检测到页面已经滚动到底部（没有更多视频可以加载）时，自动停止滚动并通知你采集完成。
- **随时中断**：在自动滚动过程中，可以随时点击**"停止采集"**按钮立即中断。

---

### 5. 📈 数据看板 Dashboard（dashboard.html + dashboard.js）
点击弹窗中的**"打开数据看板"**按钮，会在新标签页中打开一个完整的数据分析看板。

#### 📌 全局总览面板
汇总所有已采集账号的整体情况：
- **总采集视频数**：数据库中所有视频的总条数。
- **活跃账号数**：至少有 1 条视频数据的账号数量。
- **全局总粉丝数**：所有账号的粉丝数加总。
- **全局总点赞**：所有采集视频的点赞数加总（基于实际采集视频，非账号历史总数）。
- **全局总评论**：所有采集视频的评论数加总。
- **全局平均互动率**：`(总点赞 + 总评论) / 总粉丝数`，反映内容整体的粉丝互动质量。

#### 📌 账号管理面板
以表格形式列出所有被采集的账号：

| 列名 | 说明 |
|---|---|
| 账号名称 | 博主昵称（附头像） |
| 粉丝数 | 优先使用多端融合粉丝数（与抖音主页显示一致） |
| 采集视频 | 本地数据库中该账号的视频条数 |
| 总点赞 | 该账号所有**采集视频**的点赞数之和 |
| 总评论 | 该账号所有**采集视频**的评论数之和 |
| 互动率 | `(总点赞 + 总评论) / 粉丝数 × 100%` |
| 操作 | 查看详情 / 删除账号 |

#### 📌 视频排行榜面板
对所有采集视频按指定维度排序：
- 支持按**点赞数**、**评论数**、**分享数**、**收藏数**排行。
- 支持按**账号**筛选，只看某个创作者的视频排行。
- 点击视频标题可直接跳转到抖音原视频页面。

#### 📌 维护预警面板
根据视频的点赞量自动计算"预期评论数"，并与实际评论数对比：
- 若实际评论数低于预期，该视频会被标记为**需要维护**，提醒你对该视频进行评论区的维护和互动。
- 帮助运营团队快速定位哪些视频互动数据异常，需要重点关注。

#### 📌 账号详情面板
点击某个账号的"查看详情"，进入该账号专属的详情页：
- 展示该账号的所有采集视频列表。
- 显示每个视频的完整互动数据。
- 支持单独导出该账号的 CSV 文件。

---

### 6. 📤 一键导出 CSV（dashboard.js）
- **导出全部**：将所有账号的所有采集视频，按账号分组，一次性导出为一个 CSV 文件，文件名包含日期。
- **按账号导出**：在账号详情页中，可以单独导出该账号的所有视频数据。
- **UTF-8 BOM 编码**：导出文件包含 BOM 头，在 Excel 中直接打开不会出现中文乱码。

导出的 CSV 包含以下字段：
`账号ID、账号名称、视频URL、发布时间、类型、标题、点赞、评论、分享、收藏`

---

### 7. 🔄 重复采集自动更新数据
对同一个账号重复进入主页并向下滑动，插件会自动以最新数据更新数据库中的视频记录（点赞、评论等数字刷新为当前值），而不会产生重复条目。

## 🏗 技术架构

```
douyin_skill/
├── extension/
│   ├── manifest.json       # 插件配置文件（Manifest V3）
│   ├── popup.html          # 弹窗 UI 页面
│   ├── popup.js            # 弹窗交互逻辑（采集按钮、计数显示）
│   ├── content.js          # 内容脚本（自动滚动逻辑，注入页面执行）
│   ├── injected.js         # 深度注入脚本（Hook XHR/Fetch，拦截抖音 API 数据）
│   ├── background.js       # 后台 Service Worker（IndexedDB 读写、消息路由）
│   ├── dashboard.html      # 数据看板 HTML 页面
│   ├── dashboard.js        # 数据看板所有渲染逻辑
│   └── js/
│       └── toast.js        # 轻量级消息提示组件
└── README.md
```

**数据流向：**
```
抖音网页 API → injected.js (拦截) → postMessage → content.js → 
chrome.runtime.sendMessage → background.js (写入 IndexedDB)
     ↑
popup.js (GET_STATS) / dashboard.js (读取所有数据展示)
```

## 🚀 安装步骤

> **支持的浏览器**：Microsoft Edge、Google Chrome（及所有基于 Chromium 的浏览器）

### 第一步：下载项目
点击本页右上角绿色的 **Code** 按钮 → **Download ZIP**，下载后解压到任意文件夹。

### 第二步：打开浏览器扩展页面
- **Edge**：在地址栏输入 `edge://extensions/` 并回车
- **Chrome**：在地址栏输入 `chrome://extensions/` 并回车

### 第三步：开启开发者模式
在扩展页面的右上角（Edge）或右上角（Chrome），找到并打开 **"开发人员模式"** (Developer Mode) 开关。

### 第四步：加载插件
点击 **"加载解压缩的扩展"** (Load unpacked) 按钮，在弹出的文件选择框中，选择你刚才解压出来的项目文件夹里的 `extension` 子文件夹，点击确认。

### 第五步：固定插件图标（可选但推荐）
在浏览器右上角点击拼图块 🧩 图标，找到本插件，点击图钉 📌 将其固定到工具栏，方便随时使用。

## 📖 使用操作指南

1. **打开抖音网页版**：在浏览器中打开 [https://www.douyin.com](https://www.douyin.com) 并登录你的抖音账号。

2. **进入目标创作者主页**：搜索或点击进入你想分析的博主主页（URL 格式为 `douyin.com/user/xxxx`）。

3. **开始采集**：点击浏览器右上角的**插件图标**，在弹窗中点击红色的 **"自动向下滚动采集"** 按钮。
   - 如果你不想自动滚动，也可以**手动慢慢向下滑动鼠标滚轮**，插件会实时拦截页面加载出来的每一批视频数据。

4. **等待采集完成**：弹窗上方的数字会实时增长，显示本次新采集的视频数量。当数字停止增长时，说明该账号的所有视频已全部加载完毕。

5. **采集更多账号**：关闭当前主页，进入下一个博主的主页，重复步骤 2-4。

6. **打开看板查看数据**：点击弹窗中的 **"打开数据看板 (Dashboard)"** 按钮，即可查看所有已采集账号的详细数据分析。

7. **导出数据**：在看板中点击 **"导出全部 CSV"** 按钮，将数据保存为 Excel 可直接打开的表格文件。

## ❓ 常见问题

**Q：为什么弹窗上的"本次运行新抓取视频"数字从 0 开始？**
A：这是正常的！每次打开弹窗时，数字都会重置为 0，只统计"这一次打开弹窗后"新入库的视频数量，让你清楚地知道每一轮操作的成果。

**Q：为什么数据看板的点赞数和抖音主页上显示的"获赞"数字不一样？**
A：抖音主页的"获赞"是该博主账号历史生涯所有视频（包括已删除、已隐藏的视频）的总点赞量。看板里显示的是你**实际采集到的视频**的点赞数之和，通常略低于抖音主页的数字，但这个数字用于计算互动率才是科学准确的。

**Q：数据会上传到你的服务器吗？**
A：不会。所有数据 100% 保存在你自己电脑的浏览器 IndexedDB 中，与任何外部服务器完全隔离。开发者无法以任何方式访问你的数据。

**Q：会不会导致我的抖音账号被封禁？**
A：本插件的数据采集完全基于你的**正常人工浏览行为**触发，不存在任何模拟点击或自动化爬虫。使用自动滚动功能时，请注意滚动速度不要设置过快，并在合理的时间段内使用，以降低可能的平台风控风险。

**Q：重复进入同一个博主的主页会产生重复数据吗？**
A：不会。插件在每次写入视频时都会先校验该视频 ID 是否已存在，重复采集只会**更新**已有视频的最新数据（如新增的点赞量），不会产生重复条目。
